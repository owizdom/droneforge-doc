The Nimbus stack ships from five repositories, each with its own release cadence. This page summarizes notable changes; for the complete history, the GitHub commit log on each repo is authoritative.

## How `df_version` works

`df_version` is the single number that tells you whether a flashed Nimbus + drone are consistent. When the wire format, a hardware target's pinout, or an agent-runtime input variable changes, `df_version` is bumped. The desktop app surfaces a green check when both sides match the version it was built against.

See [Versioning](/firmware/versioning) for how the host consumes the frame.

## How to be notified

- Watch each repo on GitHub (Releases or All activity).
- The desktop app surfaces a banner when a `df_version` mismatch is detected after a flash; that's the practical signal you need to update.
- Subscribe to [@thedroneforge](https://x.com/thedroneforge) for product-level announcements.
